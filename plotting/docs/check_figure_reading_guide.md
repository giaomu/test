# SHAKTI 检查图逐图阅读说明

本文档对应两个脚本：

- `check_bad_nodes_mesh_size.m`：检查鳞片状水头点和网格大小、局部水头跳变的关系。
- `check_flake_mesh_regression.m`：用 logistic 回归判断网格大小及几何变量是否提高鳞片点出现概率。

## 共同定义

### 鳞片点是什么

鳞片点不是“水头很高的点”，而是“和周围邻居差得很离谱的点”。

程序对每个节点找一圈相邻节点，然后计算：

```text
局部偏离 = 本节点的水头指标 - 邻居节点中位数
```

如果 `use_excess_for_flake = true`，水头指标用：

```text
head_excess = head - overburden_head
```

也就是先扣掉冰覆压力水头背景，再看这个点相对邻居是否突兀。这样比直接看 `head` 更适合识别数值振荡。

### 峰值点和低谷点

- 峰值点：本节点比邻居中位数高很多。
- 低谷点：本节点比邻居中位数低很多。
- 鳞片点：峰值点和低谷点的并集。

直白说，如果水头场里出现“高-低-高”或“低-高-低”的棋盘状结构，这个判据会把它抓出来。

### 阈值怎么看

回归脚本里当前写法是：

```matlab
flake_abs_threshold_m = 100;
flake_percentile = 10;
flake_threshold_m = max(flake_abs_threshold_m, percentile_threshold_m);
```

最终阈值取两者中较大的那个。

注意：`prctile(x, 10)` 是第 10 百分位，不是“最大的 10%”。如果你想取局部偏离最大的 10%，应设置为：

```matlab
flake_percentile = 90;
```

如果想只看最极端 2%，应设置为：

```matlab
flake_percentile = 98;
```

## check_bad_nodes_mesh_size.m

这个脚本主要回答：鳞片点是不是和粗网格、局部水头突跳有关。

### 图 00：鳞片点计数

子图 1：鳞片点数量

- 显示峰值点、低谷点、总鳞片点数量。
- 峰值和低谷数量都多，说明水头场不是单纯整体偏高，而是存在上下交替振荡。
- 只有峰值多或只有低谷多，说明异常更偏向单侧突跳。

子图 2：鳞片点占全域比例

- 显示鳞片点占全部节点的百分比。
- 比例很低，通常是局部异常。
- 比例达到几个百分点，说明水头场已经有较系统的局部振荡。

### 图 01：水头场与鳞片点总览

子图 1：水头 `HydrologyHead [m] + 鳞片点`

- 背景是模型求出的水头。
- 红点是鳞片点。
- 用来看异常点是不是落在水头高值区、低值区、边界附近或粗糙斑块附近。
- 如果红点正好在“颜色突然翻转”的地方，说明局部水头场不连续感很强。

子图 2：超压水头 `head - 冰覆压力水头 [m] + 鳞片点`

- 背景是扣除冰覆压力水头后的剩余水头。
- 这张图更接近“异常水头”的空间图。
- 如果红点在这张图上更集中，说明异常不是正常冰厚背景造成的，而是局部水头偏离。

子图 3：水头 / 冰覆压力水头 + 鳞片点

- 背景是水头相对冰覆压力水头的比例。
- 比值接近 1，约等于接近浮力/冰覆压力水平。
- 比值大于 1 甚至远大于 1，说明水头超过冰覆压力，物理上可疑。
- 如果鳞片点也集中在高比值区，说明数值振荡和超压异常重合。

子图 4：峰值点和低谷点分类

- 通常用不同符号或颜色区分峰值点、低谷点。
- 如果峰值点和低谷点交错出现，就是典型鳞片状振荡。
- 如果只是一团峰值点，可能更像局部源项、边界或几何突变导致的高水头区。

### 图 02：鳞片强度诊断

子图 1：局部水头偏离 `head - 邻居中位数 [m]`

- 看原始水头相对邻居突兀程度。
- 正值大表示这个点比邻居高很多。
- 负值大表示这个点比邻居低很多。
- 如果正负斑块交替，说明解在节点尺度振荡。

子图 2：局部超压水头偏离 `[m]`

- 看 `head - overburden_head` 相对邻居突兀程度。
- 这是定义鳞片点时更推荐看的量。
- 如果这张图也强烈鳞片化，说明问题不是单纯地形背景，而是水头相对压力背景也在振荡。

子图 3：最大邻居水头跳变 `[m]`

- 对每个节点，找它和所有邻居之间最大的水头差。
- 用来看“相邻两个节点之间最大能跳多少”。
- 如果这个量很大，说明有限元相邻节点之间已经有很强的不连续感。

子图 4：最大邻居超压水头跳变 `[m]`

- 和子图 3 类似，但看的是超压水头。
- 如果它和鳞片点位置高度重合，说明鳞片不是视觉错觉，而是节点间真实数值跳变。

子图 1：单元 `sqrt(area) [m] + 鳞片点`

- 背景是每个三角形单元面积开方，可理解为单元典型长度。

- 它比 `sqrt(area)` 更能暴露狭长或局部很差的单元。
- 如果这里和鳞片点更重合，说明网格形状或最长边可能比面积更关键。

子图 3：节点邻域平均 `sqrt(area) [m] + 鳞片点`

- 把节点周围相邻单元的典型长度平均到节点上。
- 这是后续回归里默认使用的节点网格尺度。
- 适合判断“这个节点周围整体是不是粗”。

子图 4：节点邻域最大边长 `[m] + 鳞片点`

- 把节点周围相邻单元的最大边长统计到节点上。
- 适合抓局部最差网格。
- 如果鳞片点在这张图上更明显，说明局部最差单元可能触发振荡。

子图 2：最大边长 vs 局部水头偏离

- 横轴：节点邻域最大边长。
- 纵轴：局部水头偏离。
- 如果比子图 1 关系更强，说明最长边或局部坏单元比平均网格尺度更重要。

子图 3：网格尺度 vs 局部超压水头偏离

- 横轴：节点邻域平均 `sqrt(area)`。
- 纵轴：局部超压水头偏离。
- 这是更贴近鳞片判据的散点图。
- 关系越明显，越支持“粗网格导致相对压力背景的水头振荡”。

子图 4：最大边长 vs 局部超压水头偏离

- 横轴：节点邻域最大边长。
- 纵轴：局部超压水头偏离。
- 用来判断“最差边长”是否和异常更相关。

子图 1：普通点与鳞片点的网格尺度分布

- 比较普通点和鳞片点的 `sqrt(area)` 分布。

子图 2：普通点与鳞片点的最大边长分布

- 比较普通点和鳞片点的最长边分布。

子图 3：普通点与鳞片点的水头偏离分布

子图 4：普通点与鳞片点的超压水头偏离分布

## check_flake_mesh_regression.m

这个脚本主要回答：在控制几何、gap、速度后，网格大小是否仍能独立解释鳞片点概率。

### 图 01：分箱概率与 logistic 回归曲线

这个图通常只有一个子图。

图中元素：

- 黑色分箱点：按网格大小分箱后，每个箱子里的实际鳞片点比例。
- 误差线：这个箱子里鳞片比例的不确定范围。
- 一条回归曲线：只看网格大小时，预测的鳞片概率。
- 另一条回归曲线：控制几何、gap、流速后，只改变网格大小时的预测鳞片概率。

怎么看：

- 黑点随网格变粗上升：原始数据支持粗网格更容易出鳞片点。
- 控制变量后曲线仍上升：网格大小本身仍有独立影响。
- 控制变量后曲线变平：粗网格效应可能主要来自它和陡坡、高基岩、薄冰等区域重合。
- 黑点很乱、误差线很长：样本少或变量关系弱，不能过度解释。

### 图 02：控制变量 logistic 回归系数

这个图通常只有一个子图。

图中元素：

- 纵轴：进入回归的变量。
- 横轴：每增加 1 个标准差，鳞片点 odds ratio 改变多少。
- 红点：估计值。
- 横线：置信区间。
- 竖虚线 1：没有影响的基准线。

怎么看：

- odds ratio > 1：变量越大，鳞片点概率越高。
- odds ratio < 1：变量越大，鳞片点概率越低。
- 置信区间跨过 1：这个变量影响不稳定。
- `网格大小` 在控制变量后仍大于 1 且不跨 1，说明粗网格独立影响较强。

注意：这只是统计相关，不是单独证明因果。

### 图 03：粗网格风险比

子图 1：粗网格区自身的鳞片概率

- 横轴：粗网格分位数阈值。
- 纵轴：被归入粗网格区的节点里，有多少比例是鳞片点。
- 数值越高，说明粗网格区内部越容易出问题。

子图 2：粗网格区相对其他区域风险

- 纵轴是风险比：

```text
风险比 = 粗网格区鳞片概率 / 其他区域鳞片概率
```

- 风险比大于 1：粗网格更危险。
- 风险比接近 1：粗网格和其他区域差不多。
- 风险比小于 1：粗网格区反而不是主要问题区。

### 图 04：粗网格数量统计

子图 1：鳞片点数量：粗网格区 vs 其他区域

- 看不同分位数阈值下，鳞片点有多少落在粗网格区，多少落在其他区域。
- 如果粗网格区鳞片点数量多，说明问题不只是概率高，而且数量上也重要。

子图 2：节点数量：粗网格区 vs 其他区域

- 看不同分位数阈值下，粗网格区本身包含多少节点。
- 要和子图 1 配合看。
- 如果粗网格区节点很少但鳞片点很多，说明粗网格高度集中地产生异常。

### 图 05：几何与 gap 控制变量空间背景

这个图是 3 行 4 列布局，目前实际使用 10 个子图。

子图 1：回归主变量，节点邻域平均 `sqrt(area) [m]`

- 看网格大小空间分布。
- 鳞片点如果集中在大值区，说明粗网格是重要嫌疑。

子图 2：基底高程 `[m]`

- 看鳞片点是否集中在高基岩或深槽。
- 高基岩区容易和薄冰、边界、强坡度共同出现，需要结合子图 4、5 看。

子图 3：表面高程 `[m]`

- 看鳞片点是否集中在高表面或表面突变区。
- 如果和表面坡度强区一致，说明几何坡度可能影响水力势梯度。

子图 4：冰厚 `[m]`

- 看鳞片点是否集中在薄冰或厚冰区。
- 薄冰区水头比值容易异常，厚冰区如果也鳞片化，更多考虑网格或水力传导项。

子图 5：基岩坡度 `|grad base| [m/m]`

- 看基岩陡坡是否与鳞片点重合。
- 基岩坡度大时，水力势梯度里地形项变化快，粗网格会更难解析。

子图 6：表面坡度 `|grad surface| [m/m]`

- 表面坡度影响冰覆压力水头背景。
- 如果鳞片点沿表面坡度大区域排列，可能是压力背景变化太快。

子图 7：冰厚坡度 `|grad thickness| [m/m]`

- 冰厚坡度影响 overburden head。
- 如果它和鳞片点重合，说明扣除冰覆压力背景后仍可能受几何分辨率影响。

子图 8：gap 高度或 `log_gap高度`

- gap 影响 transmissivity，进而影响水头扩散能力。
- 小 gap 区水力传导弱，容易产生陡梯度。
- 大 gap 区可能代表通道化或局部开腔增强。
- 但 gap 是模型状态量，不是完全外生变量，解释要谨慎。

子图 9：冰流速或 `log_冰流速`

- 看鳞片点是否与高速流动区重合。
- 高速区可能对应强滑动、强开腔或高水量输入背景。
- 速度是辅助解释变量，不能单独定因果。

子图 10：响应相关量，`|局部超压水头偏离| [m]`

- 这是鳞片强度本身的空间分布。
- 用来和前 9 个物理场对比。
- 如果它只和网格大小重合，数值离散嫌疑更强。
- 如果它和坡度、gap、速度也重合，说明可能是物理强梯度和网格不足共同导致。

### 图 06：回归预测诊断

子图 1：预测概率校准检查

- 横轴：模型预测的鳞片概率。
- 纵轴：实际分箱统计出的鳞片概率。
- 红虚线附近表示预测和实际一致。
- 点明显偏离红虚线，说明回归模型校准不好。

子图 2：每个节点的预测概率

- 横轴：网格大小。
- 纵轴：回归模型给每个节点的预测鳞片概率。
- 颜色通常表示该节点是否实际为鳞片点。
- 如果实际鳞片点集中在高预测概率区域，模型有解释力。
- 如果预测概率和实际鳞片点混在一起，说明当前变量解释能力有限。

## single_variable_effects 子文件夹

每个被选择的变量会有一个独立子文件夹。

### 单变量图 01：单变量影响概率曲线

这个图通常只有一个子图。

图中元素：

- 黑点：该变量分箱后的实际鳞片概率。
- 误差线：分箱比例的不确定性。
- 红线：控制其他变量后，只改变当前变量的回归预测概率。

怎么看：

- 红线上升：该变量越大，鳞片概率越高。
- 红线下降：该变量越大，鳞片概率越低。
- 红线平：该变量独立影响弱。
- 黑点和红线差距大：该变量单独趋势和控制变量后的趋势不同，说明有混杂因素。

### 单变量图 02：普通点与鳞片点分布

这个图通常只有一个子图。

- 灰色：普通点。
- 红色：鳞片点。

怎么看：

- 红色分布明显比灰色靠右：鳞片点更偏向该变量的大值。
- 红色分布明显比灰色靠左：鳞片点更偏向该变量的小值。
- 两者高度重合：该变量单独解释能力弱。
- 红色只在极端尾部出现：该变量可能存在阈值效应。

## 当前程序是否写得对

总体逻辑是合理的：

- 鳞片点用邻域中位数偏离定义，适合抓节点尺度振荡。
- 同时看原始空间图、散点图、直方图、分位数覆盖率、logistic 回归，逻辑是完整的。
- 回归里没有把 `HydrologyBasalFlux` 当作控制变量是对的，因为水通量直接由水头梯度强相关计算出来，容易把水头异常本身解释掉。
- `gap` 和冰流速作为辅助变量可以用，但解释时要谨慎。

需要特别注意的地方：

- `flake_percentile = 10` 如果你的意思是“最极端 10%”，那应该改成 `90`。
- 如果输出里有效样本数为 0，优先检查某个控制变量是否全是 NaN，尤其是 gap 或冰流速。
- 如果 `logistic` 出现奇异矩阵，通常说明变量强共线、某个变量几乎常数、或者有效样本不足。
